GEO数据解读与存储

GEO数据库,全称为GENE EXPRESSION OMNIBUS ,是由NCBI创建并维护的基因表达数据库 。

基本概念

  • GEO Platform (GPL)

    代表的是测序平台或者芯片平台，它描述了一连串在特定实验中被检测或被定量分析的因素。每一个platform使用GPL开头的编号唯一标识，每一个platform，通常包含3种文件,即soft、miniml和suppl

    soft和miniml表示该platform的基础信息，如GPL编号和上传日期等。通常每种类别的信息以^开头，常见的类别有Database,Platform,Sample, Series；在每种类别中，!开头代表一种类型的信息。

    对于芯片平台还会提供探针和基因ID间的关系，即芯片ID的基因注释信息。

    miniml中内容和soft一致，只是用XML格式存储

    suppl是supplement file，是提交者自己上传的补充文件。

  • GEO Sample (GSM)

    代表的是一个以平台为基础、描述某个杂交实验或者实验条件的所有特征因素的大量测量信息。即被检测的mRNA的样本，实验条件和实验产生的基因表达测量数据信息。

    每个样本都有一个特有的检索号GSM。

  • GEO Series (GSE)

    系列是指把构成某个实验的相关样本集中到一个有生物学意义的数据集，同时可能还收集一些已被提交者注明的重要基因或者分析结果，包括样本收集、如何相关联、如何排序、如何分析和如何获得聚类信息。

    一个系列里的样品是通过某一共同属性联系在一起的

  • GEO Dataset (GDS)

    GEO中包含一个辅助分析工具，可以把被提交的样本归纳分类到有生物学意义和在统计学上可比较的GEO数据集组，以方便为下游数据挖掘等。

    总之，一篇文章可以有一个或者多个GSE数据集，一个GSE里面可以有一个或者多个GSM样本。多个研究的GSM样本可以根据研究目的整合为一个GDS，不过GDS本身用的很少。而每个数据集都有着自己对应的芯片平台，就是GPL。

GEOquery包从GEO数据库下载数据

  1. 下载和加载包

    source("http://www.bioconductor.org/biocLite.R")
biocLite("GEOquery")
library(GEOquery)

    通过bioconductor包来获取所有的芯片探针与gene的对应关系

  2. 下载函数并使用—主要是getGEO()函数

    • 根据GDS号下载数据、soft文件

      gds858 <- getGEO(‘GDS858’, destdir=“.”)

    • 根据GPL号下载的是芯片设计的信息 

      gpl96 <- getGEO(‘GPL96’, destdir=“.”)

    • 根据GSE号下载数据，下载_series_matrix.txt.gz 

      gse1009 <- getGEO(‘GSE1009’, destdir=“.”)

      文件保存在本地，destdir指定了下载地址

  3. 函数返回的对象

    • 查看下载GDS后返回的对象

      options(warn=-1)
suppressMessages(library(GEOquery))
gds858 <- getGEO('GDS858', destdir=".")
names(Meta(gds858))#Meta()可以得到描述信息
Table(gds858)[1:5,1:5]#得到表达矩阵

      note：GDS2eSet ()函数可以把它转换为expression set 对象！

      eset <- gds2eset(gds858, do.log2="TRUE)" < code>

    • 查看下载GSE后返回的对象

      就是expression set 对象！

      处理函数有geneNames(), sampleNames(), pData, expers

    • 查看下载GPL后返回的对象

      与GDS同，使用Table()和Meta()处理

      Table(gpl96)[1:10,c("ID","GB_LIST","Gene.Title","Gene.Symbol","Entrez.Gene")]#可获得芯片ID的基因注释信息

包装成函数

downGSE <- function(studyID = "GSE1009", destdir = ".") {

    library(GEOquery)
    eSet <- getGEO(studyID, destdir = destdir, getGPL = F)

    exprSet = exprs(eSet[[1]])
    pdata = pData(eSet[[1]])

    write.csv(exprSet, paste0(studyID, "_exprSet.csv"))
    write.csv(pdata, paste0(studyID, "_metadata.csv"))
    return(eSet)

}#下载GEO数据集的表达矩阵过程，写成专门的函数
