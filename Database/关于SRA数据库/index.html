关于SRA数据库

SRA（Sequence Read Archive）数据库是用于存储二代测序的原始数据，包括 454，Illumina，SOLiD，IonTorrent，Helicos 和 Complete Genomics。除了原始序列数据外，SRA现在也存在raw reads在参考基因的比对信息。

分类

根据SRA数据产生的特点，分为四类

  • Studies 研究课题(ERP or SRP)

    就实验目标而言，可包含多个Experiments

  • Experiments 实验设计(SRX)

    包含了Sample、DNA source、测序平台、数据处理等信息。可包含一个或者多个RUN

  • Runs 测序结果集(SRR)

    表示测序仪运行产生的reads

  • Samples 样品信息(SRS)

SRA中数据结构的层次关系为：SRP(项目)—>SRS(样本)—>SRX(数据产生)—>SRR(数据本身)

使用

  • 在NCBI里选择SRA数据库，查询SRP开头的ID，可以得到数据集里面数据信息。（一般文章如果将提到的数据上传到了SRA数据库会给出SRP开头的ID，如SRP078156）

  • 使用SRA Run Selector搜索相应的SRP号，可以获得project里的数据信息，数据大小。

  • BioProject:PRJNA327548 电击这个PRJNA开头的ID，就可以获得发表文章、样本数量等信息。等同于在BioProject里搜索SRP开头的ID。

  • 进入每个样成本，等同于SRS开头的ID，可以根据find related data 以及related information按钮搜索到样本的信息，可以知道每个样本有几个数据，iterm=

  • 点开任意一个数据，就进入了SRX开头的每个数据的详细描述

  • 在这里可以点开SRR开头的数据本身，即RUNs包含的信息。

数据下载

SRR开头的runs的数据是可以下载并进行操作的，下载需要先安装SRA Toolkit软件包 。根据环境下载相应的软件包，下载数据后转换fastq，转换fasta格式等操作。（还需要掌握具体方法）

数据提交

SRA study 所包含的内容， 应该在一个 LSBI 的项目中提交 。一个完整的 study， 包括一个或多个 study.xml, experiment.xml, sample.xml 和 run.xml， 以及一个或多个数据文件。 但是一个批次的提交数据不一定包括所有的文件。
